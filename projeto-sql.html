Exemplo de consulta · Módulo 04 — GROUP BY

04_group_by / 02_group_by.sql
1-- Top 10 clientes que mais fizeram pontos no mês de Julho/2025
2SELECT t.idCliente,
3 SUM(t.qtdPontos) AS total_pontos
 4FROM transactions t
5WHERE strftime('%Y-%m', t.dtTransaction) = '2025-07'
 6GROUP BY t.idCliente
7ORDER BY total_pontos DESC
8LIMIT 10;
01 A Importância do SQL

Por que SQL é
fundamental.

SQL é a linguagem universal dos dados. Independente da ferramenta que você usa — Power BI, Python, Tableau ou qualquer plataforma de BI — a capacidade de interrogar bancos de dados diretamente é o que diferencia um analista que depende de extrações prontas de um profissional que consegue acessar, transformar e validar os dados na origem.

Para um Analista de Dados e BI, dominar SQL significa ter autonomia total no ciclo de trabalho: sem aguardar times de engenharia para extrações específicas, sem depender de planilhas intermediárias e com capacidade de validar a qualidade dos dados antes de qualquer visualização.

Acesso direto aos dados

Consultar bancos de dados diretamente — sem intermediários — é o que permite análises mais profundas, rápidas e confiáveis em qualquer ambiente corporativo.

Validação e qualidade de dados

Identificar duplicatas, nulos, outliers e inconsistências na fonte é uma competência crítica — e SQL é a ferramenta mais eficiente para isso.

Fundamento para ferramentas avançadas

Python com Pandas, DAX no Power BI e Spark seguem a mesma lógica de SQL. Quem domina SQL aprende qualquer ferramenta de dados com muito mais facilidade.

Linguagem do mercado

Presente em praticamente 100% das vagas de dados e BI. SQL é um requisito universal — do analista júnior ao engenheiro sênior de dados.

03 Capacidade Demonstrada

O que esse estudo
demonstra.

Um repositório organizado, documentado e progressivo diz muito mais sobre um profissional do que uma lista de cursos no currículo. Veja o que este projeto evidencia na prática:

  • 01
    Disciplina e método de aprendizado

    Cada exercício foi documentado com a pergunta de negócio que responde — demonstrando que o aprendizado foi intencional, orientado a resultados e não apenas à sintaxe.

  • 02
    Raciocínio analítico aplicado

    Os exercícios avançam de consultas simples para problemas compostos — evidenciando a capacidade de quebrar questões complexas em etapas lógicas de consulta.

  • 03
    Uso de base de dados real

    Trabalhar com o dataset teomewhy-loyalty-system do Kaggle — com tabelas de clientes, transações e produtos reais — é muito mais valioso do que exemplos genéricos, pois simula o ambiente corporativo.

  • 04
    Complemento direto ao trabalho em BI

    SQL e Power BI se complementam: SQL valida e prepara os dados na fonte; o Power BI transforma esses dados em visualização. Dominar os dois é o perfil buscado pelo mercado de BI.

  • 05
    Evolução contínua e documentada

    O repositório registra a progressão real do aprendizado — de WHERE básico a GROUP BY com HAVING — mostrando não apenas onde estou, mas o caminho percorrido para chegar aqui.
